# 申论批改流程

> 用法：以后你把申论答案发给我，我就按这套固定流程来，不随口说“还行”或者“不错”。
> 目标：把申论批改变成标准化动作，而不是靠感觉评价。

---

## 一、先判断属于哪类题

收到题目后，先分类型：

- [ ] 概括题
- [ ] 对策题
- [ ] 分析解释题
- [ ] 公文题
- [ ] 大作文

不同题型，用不同标准看，不能混着评。

---

## 二、先拆题干，不急着看答案

固定先看这几项：

1. 题目让我做什么
2. 让我以什么身份写
3. 写给谁看
4. 有没有文种要求
5. 字数要求是什么
6. 是概括、分析、提出对策，还是写文章

如果这一步没拆对，后面分再高也不稳。

---

## 三、再看你的答案是否回应了题目

先不急着评价文采，先判断：

- 有没有直接回答题目要求
- 有没有偏离任务
- 有没有漏掉主要任务
- 有没有把一种题写成另一种题

例子：
- 概括题写成评论文
- 对策题全在分析原因
- 公文题没有对象和语气
- 大作文全是套话，没有中心

---

## 四、按固定维度批改

以后统一按下面顺序：

### 1. 审题
- 方向对不对
- 任务有没有回应全

### 2. 要点
- 大点有没有漏
- 有没有抓住材料关键内容

### 3. 结构
- 有没有分层
- 段落安排顺不顺
- 开头、主体、结尾是否清楚

### 4. 语言
- 是否像申论语言
- 是否口语化、重复、空泛
- 句子是否通顺

### 5. 特殊要求
- 公文看格式、对象、语气
- 大作文看立意、论证、材料结合

---

## 五、统一输出格式

以后我批改时，固定按这个格式给你：

### 1. 题型判断
这是一道：__________

### 2. 一句话总评
这篇答案目前最大的问题是：____________________

### 3. 维度评分
- 审题：A / B / C / D
- 要点：A / B / C / D
- 结构：A / B / C / D
- 语言：A / B / C / D
- 特殊项：A / B / C / D

### 4. 最大失分点
- 失分点1：______________________________________
- 失分点2：______________________________________
- 失分点3：______________________________________

### 5. 最大保分点
- 保分点1：______________________________________
- 保分点2：______________________________________

### 6. 综合档位
- [ ] 高档
- [ ] 中上
- [ ] 中档
- [ ] 危险
- [ ] 低档

### 7. 修改动作
- 第一步先改：____________________________________
- 第二步再改：____________________________________
- 第三步补强：____________________________________

### 8. 可选示范
- 我可以只给你改开头
- 或只给你补漏点
- 或只给你示范一段
- 不默认整篇重写

---

## 六、批改时我不做的事

为了避免误导，以后不做这几件事：

1. 不随口说“这篇能上多少分”
2. 不因为语言顺就误判成高分
3. 不脱离题目和材料空谈写作技巧
4. 不把“看着舒服”当成“能拿高分”

---

## 七、批改时最看重的顺序

重要性从高到低，一般是：

1. 是否跑题
2. 是否漏大点
3. 是否回应题目任务
4. 是否有清晰结构
5. 语言是否规范

也就是说：

**先保方向，再保要点，再谈表达。**

---

## 八、最重要的一句话

**申论批改不是看你写得像不像文章，而是看你有没有按题目要求把该得的分写出来。**
